Python 3.10+ Vue 3 FastAPI LangGraph Apache 2.0

RAG 智库系统 +
AIOps 运维诊断 Agent

企业级 RAG 知识库问答系统,并在其上构建了一条基于 LangGraph 的 AIOps 运维诊断 Agent 支线 —— 前者支持多格式文档解析、向量化、智能问答与自动评测; 后者通过 Plan-Execute-Replan 工作流 + MCP 协议工具,演示 Agent 自主故障诊断。

⭐ 在 GitHub 上查看 了解核心特性 ↓

核心特性

从知识库问答到自主运维诊断,一套底座两条链路

📚

多模态知识库

支持 PDF / Word / Excel / PPT / Markdown 等格式,内置 OCR 与结构分析,上传后自动异步切分与向量化。

🤖

智能对话助手

可自定义系统提示词、温度与模型;短期记忆滑动窗口;混合检索 + 重排序;回答可溯源到具体段落与页码。

🔧

AIOps 诊断 Agent

基于 LangGraph 的 Plan-Execute-Replan 工作流,通过 MCP 协议调用监控 / 日志工具,自主完成故障诊断。

📊

RAG 自动化评测

自动生成单跳 / 多跳 QA 对,结合 RAGAS 思想给出忠实度、相关性、召回率多维打分与可视化报告。

🛡️

企业级架构

RBAC 权限;FastAPI + Celery + Milvus + MinIO + Redis 高可用组合;多用户隔离。

🪂

优雅降级

Milvus 不可用降级本地文件检索,MCP 不可达降级 LLM 知识分析 —— 任一组件故障都不阻断主流程。

系统架构

RAG 与 AIOps 共享底座,编排相互独立

                          ┌─────────────┐
     Vue3 前端 ──HTTP──►   │   FastAPI    │
                          └──────┬──────┘
              ┌──────────────────┼──────────────────┐
              ▼                  ▼                  ▼
        RAG 问答链路        AIOps Agent         异步任务
     (retrieval+rerank)   (LangGraph 图)      (Celery)

                          (monitor / cls)

技术选型

每个选择背后都有工程权衡

LangGraph StateGraph

而非 ReAct 自由循环 —— 诊断任务需收敛出报告,StateGraph + 硬限制保证终止,显式节点便于扩展决策。

MCP 协议解耦工具

而非 @tool 写死 —— 工具作为独立服务,客户端只认 URL,Mock → 真实后端零改动 Agent。

双 LLM 路径

Agent 侧需 function calling 走 ChatOpenAI 兼容模式,RAG 侧走 ChatTongyi 原生 SDK,各取所长。

FastAPI Vue 3 LangGraph Milvus Celery + RabbitMQ MinIO Redis DashScope / 通义千问 MCP

快速开始

四步在本地跑起来

1

启动基础服务

cd docker && docker-compose up -d
Milvus / MinIO / Redis / RabbitMQ

2

后端部署

pip install -r requirements.txt
python create_tables.py
python -m src.main

3

前端部署

cd frontend && npm install
npm run dev

4

AIOps 诊断(可选)

bash scripts/demo.sh
一键启动 MCP 指标 + 日志 + 后端
