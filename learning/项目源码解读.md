# 项目源码解读

本文档详细介绍了 `ragPdfSystem` 项目的核心模块实现方案，包括新增的知识库管理、权限系统及评测模块。

## 1. 系统架构升级

项目从单一的 RAG 脚本升级为前后端分离的企业级应用：
- **前端**: Vue 3 + Element Plus，提供可视化的管理界面。
- **后端**: FastAPI + SQLAlchemy，引入了关系型数据库 (SQLite) 管理元数据。
- **存储**: 
  - **Milvus**: 存储向量数据 (新增 `kb_id` 字段支持多知识库隔离)。
  - **SQLite**: 存储用户、知识库、文档状态、对话历史、QA 对及评测结果。

## 2. 核心模块详解

### 2.1 数据库层 (`src/database/`)
- **SQLAlchemy Models (`models.py`)**: 定义了 `User`, `KnowledgeBase`, `KnowledgeDocument`, `ChatSession`, `GeneratedQAPair`, `EvaluationTask` 等表结构。
- **Milvus Client (`vector_db.py`)**: 升级了 Schema，增加了 `kb_id` 标量字段，并在检索时支持 `expr` 过滤，实现知识库隔离。

### 2.2 接口层 (`src/api/routers/`)
- **Auth (`auth.py`)**: 实现 JWT 登录与注册。
- **Knowledge Base (`knowledge_base.py`)**: 
  - 管理知识库的 CRUD。
  - 处理文件上传，触发后台解析任务。
  - **QA 生成**: 调用 `qa_generator` 服务，利用 LLM 从文档中提取标准问答对并入库。
- **Chat (`chat.py`)**: 
  - 管理会话 (`ChatSession`) 和消息 (`ChatInteraction`)。
  - 调用 `RAGService` 进行问答，并自动保存历史记录。
- **Evaluation (`evaluation.py`)**: 
  - 启动后台评测任务。
  - 生成并提供 Markdown 格式的评测报告下载。

### 2.3 业务逻辑层 (`src/services/`)
- **RAG Service (`rag_service.py`)**: 
  - 升级 `query` 方法，支持传入 `kb_id` 和 `session_id`。
  - 集成 `MemorySystem` 进行多轮对话上下文管理。
- **QA Generator (`qa_generator.py`)**: 
  - 封装 LLM Prompt，专门用于从文本中拆分生成 JSON 格式的 QA 对。
- **Evaluator (`evaluator.py`)**: 
  - 实现了基于 LLM 的评测逻辑，计算 Faithfulness, Relevancy, Context Precision 指标。
  - 自动生成 Markdown 汇总报告。

## 3. 前端实现 (`frontend/`)
- **Views**:
  - `Login.vue`: 用户认证。
  - `Dashboard.vue`: 功能导航。
  - `KnowledgeBase.vue`: 知识库管理与文档上传/QA生成。
  - `Chat.vue`: 类似 ChatGPT 的对话界面，支持选择知识库。
  - `Evaluation.vue`: 启动评测与查看报告。
- **State Management**: 使用 Pinia (在 `main.js` 中初始化) 或简单的 ref/reactive 管理状态。
- **API**: 统一封装 Axios 请求拦截器，自动处理 JWT Token。

## 4. 关键流程

### 4.1 多知识库检索
1. 用户在前端选择知识库 (KB ID)。
2. 请求发送至 `/api/v1/chat`，携带 `kb_id`。
3. 后端 `RAGService` 调用 `VectorRetriever`。
4. `VectorRetriever` 在 Milvus `search` 时添加 `expr="kb_id == {id}"`。
5. Milvus 仅返回该知识库下的向量片段。

### 4.2 自动化评测
1. 用户点击“开始评测”。
2. 后端从 `generated_qa_pairs` 表中抽取 N 个已审核的 QA 对。
3. 依次对每个问题运行 RAG 检索与生成。
4. 将 (问题, 标准答案, 生成答案, 检索上下文) 送入 `RAGEvaluator`。
5. LLM 打分并返回 JSON 结果。
6. 汇总所有结果生成 Markdown 报告并保存。
