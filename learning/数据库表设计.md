# 数据库表设计

本文档详细描述了 RAG 系统的数据库表结构，包括用户权限、知识库管理、问答记录、评估测试等模块。

## 1. 核心业务表

### 1.1 知识库表 (`knowledge_bases`)
存储知识库的基本信息。

| 字段名 | 数据类型 | 约束 | 描述 |
| :--- | :--- | :--- | :--- |
| `id` | BIGINT | PRIMARY KEY, AUTO_INCREMENT | 知识库ID |
| `kb_uid` | VARCHAR(64) | UNIQUE, NOT NULL | 业务唯一标识 |
| `name` | VARCHAR(128) | NOT NULL | 知识库名称 |
| `description` | TEXT | | 描述 |
| `owner_id` | BIGINT | FOREIGN KEY, NOT NULL | 创建者ID (关联 `users.id`) |
| `is_public` | TINYINT | DEFAULT 0 | 是否公开: 1-是, 0-否 |
| `vector_store_collection` | VARCHAR(128) | | 对应的向量库集合名称 (可选，若分集合存储) |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

### 1.2 知识库文档表 (`knowledge_documents`)
存储原始文档的元数据。

| 字段名 | 数据类型 | 约束 | 描述 |
| :--- | :--- | :--- | :--- |
| `id` | BIGINT | PRIMARY KEY, AUTO_INCREMENT | 文档ID |
| `doc_uid` | VARCHAR(64) | UNIQUE, NOT NULL | 业务唯一标识 |
| `kb_id` | BIGINT | FOREIGN KEY, NOT NULL | 所属知识库ID (关联 `knowledge_bases.id`) |
| `filename` | VARCHAR(255) | NOT NULL | 文件名 |
| `file_path` | VARCHAR(1024) | NOT NULL | 存储路径 |
| `file_type` | VARCHAR(16) | NOT NULL | 文件类型 (pdf, docx, etc.) |
| `file_size` | BIGINT | | 文件大小 (Bytes) |
| `status` | TINYINT | DEFAULT 0 | 状态: 0-上传中, 1-处理中, 2-已完成, 3-失败 |
| `chunk_count` | INT | DEFAULT 0 | 切分片段数 |
| `error_msg` | TEXT | | 错误信息 |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 上传时间 |

### 1.3 文档片段表 (`document_chunks`)
存储文档切分后的片段信息 (主要用于反查和展示，向量在 Milvus)。

| 字段名 | 数据类型 | 约束 | 描述 |
| :--- | :--- | :--- | :--- |
| `id` | BIGINT | PRIMARY KEY, AUTO_INCREMENT | 片段ID |
| `chunk_uid` | VARCHAR(64) | UNIQUE, NOT NULL | 业务唯一标识 |
| `doc_id` | BIGINT | FOREIGN KEY, NOT NULL | 所属文档ID |
| `content` | TEXT | NOT NULL | 片段内容 |
| `page_num` | INT | | 页码 |
| `vector_id` | VARCHAR(64) | NOT NULL | 对应 Milvus 中的 ID |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

### 1.4 生成的QA对表 (`generated_qa_pairs`)
存储从文档中拆分生成的标准 QA 对。

| 字段名 | 数据类型 | 约束 | 描述 |
| :--- | :--- | :--- | :--- |
| `id` | BIGINT | PRIMARY KEY, AUTO_INCREMENT | ID |
| `kb_id` | BIGINT | FOREIGN KEY, NOT NULL | 所属知识库ID |
| `doc_id` | BIGINT | FOREIGN KEY | 所属文档ID (可选，若跨文档则为空) |
| `question` | TEXT | NOT NULL | 问题 |
| `answer` | TEXT | NOT NULL | 答案 |
| `qa_type` | VARCHAR(32) | DEFAULT 'single_hop' | 类型: single_hop, multi_hop, summary, error |
| `status` | TINYINT | DEFAULT 0 | 状态: 0-待审核, 1-已确认, 2-已废弃 |
| `created_by` | VARCHAR(32) | DEFAULT 'system' | 创建者 (system 或 user_id) |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

## 2. 问答与评估表

### 2.1 问答记录表 (`chat_interactions`)
记录用户的问答历史。

| 字段名 | 数据类型 | 约束 | 描述 |
| :--- | :--- | :--- | :--- |
| `id` | BIGINT | PRIMARY KEY, AUTO_INCREMENT | ID |
| `session_id` | VARCHAR(64) | NOT NULL | 会话ID (用于多轮对话) |
| `user_id` | BIGINT | FOREIGN KEY, NOT NULL | 用户ID |
| `kb_id` | BIGINT | FOREIGN KEY | 关联知识库ID |
| `query` | TEXT | NOT NULL | 用户提问 |
| `answer` | TEXT | NOT NULL | 系统回答 |
| `retrieved_docs` | JSON | | 检索到的文档片段 (Snapshot) |
| `metrics` | JSON | | 性能指标 (耗时等) |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 提问时间 |

### 2.2 评估任务表 (`evaluation_tasks`)
存储 RAG 评测任务配置。

| 字段名 | 数据类型 | 约束 | 描述 |
| :--- | :--- | :--- | :--- |
| `id` | BIGINT | PRIMARY KEY, AUTO_INCREMENT | ID |
| `name` | VARCHAR(128) | NOT NULL | 任务名称 |
| `kb_id` | BIGINT | FOREIGN KEY | 关联知识库 |
| `doc_ids` | JSON | | 指定文档ID列表 (可选) |
| `config` | JSON | | 评测配置 (QA数量, 比例, LLM参数) |
| `status` | TINYINT | DEFAULT 0 | 状态: 0-未开始, 1-进行中, 2-已完成, 3-失败 |
| `report_path` | VARCHAR(1024) | | 评测报告路径 (Markdown) |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

### 2.3 评估结果详情表 (`evaluation_results`)
存储具体的评测指标。

| 字段名 | 数据类型 | 约束 | 描述 |
| :--- | :--- | :--- | :--- |
| `id` | BIGINT | PRIMARY KEY, AUTO_INCREMENT | ID |
| `task_id` | BIGINT | FOREIGN KEY, NOT NULL | 任务ID |
| `question` | TEXT | | 测试问题 |
| `ground_truth` | TEXT | | 标准答案 |
| `generated_answer` | TEXT | | 生成答案 |
| `metrics` | JSON | | RAGAS 指标 (faithfulness, answer_relevancy, etc.) |
| `latency` | FLOAT | | 响应时间 (ms) |

## 3. 用户权限表 (RBAC)

### 3.1 用户表 (`users`)
| 字段名 | 数据类型 | 约束 | 描述 |
| :--- | :--- | :--- | :--- |
| `id` | BIGINT | PRIMARY KEY, AUTO_INCREMENT | ID |
| `username` | VARCHAR(64) | UNIQUE, NOT NULL | 用户名 |
| `password_hash` | VARCHAR(255) | NOT NULL | 密码哈希 |
| `role` | VARCHAR(32) | DEFAULT 'user' | 角色: admin, user, editor |
| `is_active` | TINYINT | DEFAULT 1 | 是否激活 |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 注册时间 |

### 3.2 角色权限表 (简化版)
若需要复杂权限，可扩展 `roles`, `permissions`, `role_permissions` 表。目前建议在代码中通过 `role` 字段控制。

## 4. 助手配置表

### 4.1 助手配置 (`assistant_configs`)
| 字段名 | 数据类型 | 约束 | 描述 |
| :--- | :--- | :--- | :--- |
| `id` | BIGINT | PRIMARY KEY, AUTO_INCREMENT | ID |
| `user_id` | BIGINT | FOREIGN KEY, NOT NULL | 用户ID |
| `llm_model` | VARCHAR(64) | | 模型名称 |
| `prompt_template` | TEXT | | 提示词模板 |
| `temperature` | FLOAT | DEFAULT 0.7 | 温度 |
| `enable_rerank` | TINYINT | DEFAULT 0 | 是否开启重排序 |
| `top_k` | INT | DEFAULT 5 | 召回数量 |
| `hybrid_search_weight` | FLOAT | DEFAULT 0.5 | 混合检索权重 |
